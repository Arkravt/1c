1. Вкладка "Наборы данных"
Виды наборов данных: 
- Запрос - классический вариант 
- Объкт - может использоваться когда данные хранятся в другой базе. Описывается с помощью встроенного языка.
- Объединение - используется для объединения двух наборолв данных.

Колонка Поле - это поле информационной базы соответствует полю из запроса.
Колонка Путь - это служебное наименование поля для разработчика. Может использоваться для сопостовления разных наборов данных. Можно создавать вложенные пути. Например поместить качество товара в номенклатуру.
Колонка Автозаголовок - это то наименоваине колонки которое будет пвидеть пользователь.
Колонка с ограничениями - возможность ограничивать поля и вложенные поля ссылочных типов в отборах, в группировки, в сортировки и в выбранных полях.
Колонка Выражение представления - зденсь можно использовать разлиные выражения например, 
конкатенацию строк, полей запроса, монжо даже обращщаться к общим модулям (но этим лучше не злоупотреьблять так как эти выраджения будут исполняться для каждой строки результата отчета, что может привести к медленной работе отчета.)
Колонка Выражения упорядочивания - можно указать не стандартную сортировку. Монжо использовать разные выражения как в колонке Выражения представления.
Колонка Тип значения - для того чтобы в отчете использовался определнный тип данных если поле является составным.
Колонка Оформление - Настройка вывода колонки (цвет, рамка, формат и т.д.)
Колонка Роль - указывает, что из себя представляет данное поле. Каждая роль поля может содержать свое свойство. 
Например, свойство «Период» имеет числовое значение и содержит номер периода, если поле период. Если значение свойство «Период» равно 0 (ноль), то это означает, что данное поле периодом не является

2. Вкладка "Вычисляемые поля" — это дополнительное поле данных, значения которого вычисляются по формуле. 
Вычисляемые поля позволяют вам создавать новые измерения и показатели. При этом источник данных не изменяется.
Колонка «Путь к данным» - задает наименование поля. 
Колонка «Выражение» - прописывается формула для вычисления поля. Можно использовать арифметические выражения, конструкцию «ВЫБОР КОГДА ТОГДА КОНЕЦ», и обращаться к экспортным функциям общих модулей системы.
Вы вычисляемых полях мы не можем использовать другие вычисляемые поля.
Остальные колонки настроек такие же как во вкладке "Наборы данных".
По сути вычисляемые поля можно так же создать и в запросах, но не все поля можно создать в запросах, 
например если нам нужно использовать разные наборы данных или комбинировать данные из запрсоа и объекта, то нам нужно будет использовать только Вычисляемые поля.


3. Вкладка "Ресурсы" - здесь настраиваются итоги отчетов. 
На этой вкладке мы определеяем только ресурсы, тоесть те поля которые нужно посчитать на уровне группировок, но сами группировки настраиваются уже на уровне варианта отчета.
Колонка Выражение - указывается агрегатная функция по которой нужно получить значение ресурса. Так же можно указывать собственное произвольное выражение. Можно использовать функцию "Вычислить", чтобы задать свою формулу.
Колонка Рассчитывать по - здесь мы можем указать по каким группировкам можно рассчитывать ресурс.



4. При возможности лучше использовать отборы СКД вместо отбора на уровне запроса так как это более гибко. 
Отбор СКД будет формировать запрос с нужным условием "ГДЕ".
Отбор на уровне запроса можно делать когда нам нужно жестко отфильровать данные по какому нибудь значению(напрмер по значению перечисления) и чтобы пользователь не смог его изменить.


5. Сортировку так же правильнее наклыдватиь на уровне настройки варианта отчета, чтобы пользователь видел какая сортировка у негот включена 
и чторбы он сам смог её настраивать.
Если нужно жестко прописать сортировку пользователю, то можно ее сделать на уровне запроса и запретить пользователю её менять,
запретить менять сортировку можно через ограничения поля.


6. Чтобы настроить необязатьельный параметр виртуальной таблицы нужно указать этот параметр в конструкторе запроса на вкладке "Компановка данных" через шестеренку.
В тексте запроса он будети представлен с фигурными скобками.


7. Чтобы параметры не были доступны пользователю можно поставить галку ограничение доступности на вкладке "Параметры".


8. Вкладка "Компановка данных" в конструкторе запроса очень часто используется для переопределения параметров виртуальной таблицы (например периода).
